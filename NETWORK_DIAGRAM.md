## Traffic Flow Examples

### Example 1: User Accessing Open WebUI

```
1. User browses to: https://ai.1sourcesystems.com.au
   â”‚
2. DNS lookup â†’ Cloudflare â†’ Returns your public IP
   â”‚
3. Request hits ISP Router â†’ Port 443 forwarded to Docker Server
   â”‚
4. Traefik receives request on port 443
   â”‚
5. Traefik checks hostname: "ai.1sourcesystems.com.au"
   â”‚
6. Traefik routes to: open-webui:8080 (external network)
   â”‚
7. Open WebUI needs AI model â†’ Calls ollama:11434 (internal network)
   â”‚
8. Ollama processes request â†’ Returns to Open WebUI
   â”‚
9. Open WebUI returns response â†’ Traefik â†’ User
```

### Example 2: n8n Accessing PostgreSQL

```
1. n8n workflow needs database access
   â”‚
2. n8n connects to: postgres:5432 (internal network)
   â”‚
3. PostgreSQL processes query
   â”‚
4. Returns data to n8n
   â”‚
   Note: This traffic NEVER leaves the Docker host
   PostgreSQL is NOT accessible from internet
```

### Example 3: IP Address Changes

```
1. ISP assigns new IP to your router
   â”‚
2. cloudflare-ddns container checks public IP (every 5 minutes)
   â”‚
3. Detects IP has changed
   â”‚
4. Updates Cloudflare DNS A record via API
   â”‚
5. DNS propagates globally (1-2 minutes)
   â”‚
6. All subdomains now point to new IP
   â”‚
   Result: Zero downtime, automatic failover
```
